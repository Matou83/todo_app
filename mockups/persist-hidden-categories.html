Feature 5 — Persistance des catégories cachées · 3 propositions

Option A
localStorage — instantané, zéro backend
💾 Stockage local (navigateur)
📅 Lundi — tu caches "Perso"
Toutes Ops IA Perso
💾 Sauvegardé dans localStorage
↓ ferme l'onglet
📅 Mardi — réouverture de l'app
Toutes Ops IA Perso
✓ "Perso" toujours cachée — restaurée depuis localStorage
⚠️ Mais sur un autre appareil…
💻 Mac · caché
Perso
📱 iPhone · visible
Perso

Les catégories cachées sont sauvegardées dans le localStorage du navigateur. La préférence persiste entre les sessions sur le même appareil.

✓ Instantané, zéro requête réseau
✓ Trivial à implémenter (2 lignes)
✓ Fonctionne sans compte Supabase
✗ Non synchronisé entre appareils
✗ Perdu si on vide le cache

Option B
Supabase — synchronisé tous appareils
☁️ Stockage serveur (Supabase)
📅 Lundi — tu caches "Perso" sur Mac
Toutes Ops Perso
☁️ Sauvegardé dans Supabase (table user_settings)
↓ ouvre l'app sur iPhone
📱 iPhone — même état automatiquement
💻 Mac · synchronisé
Perso
📱 iPhone · synchronisé
Perso
🗄️ Table Supabase
user_settings
├── user_id: "abc123"
├── hidden_categories: ["perso"]
└── updated_at: "2025-05-07"

Les catégories cachées sont stockées dans une table Supabase `user_settings` (JSON). Synchronisation automatique entre tous les appareils connectés au même compte.

✓ Synchronisé Mac / iPhone / autre
✓ Persiste même si cache vidé
✗ Requête Supabase à chaque changement
✗ Nécessite une migration de schéma DB

Option C
localStorage par userId — meilleur des deux
🔑 localStorage · clé par compte
👤 Compte A — cache "Perso"
Perso Ops
hidden_cats_abc123 = ["perso"]
↓ déconnexion → Compte B
👤 Compte B — ses propres préférences
Perso Ops
hidden_cats_xyz789 = ["ops"]
✓ Avantages combinés
✓ Préférences isolées par compte
✓ Zéro backend, instantané
✓ Persiste entre sessions
✗ Pas de sync multi-appareils

Comme Option A, mais la clé localStorage inclut l'ID utilisateur : hidden_cats_{userId}. Chaque compte a ses propres préférences sur le même navigateur. Instantané, sans backend.

✓ Préférences par compte (multi-user safe)
✓ Instantané, aucune requête réseau
✓ 5 lignes de code
✗ Toujours pas synchronisé entre appareils
